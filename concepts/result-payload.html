Concepts · v1

TransactionResult payload

TransactionResult is the curated, normalised view of a successful terminal-state transaction. Every field, when it is populated, transport variations, and how to use the most-frequently-misunderstood one — chargeSlipPrintData.

01

Field reference

Fields marked Always are guaranteed on every onSuccess callback. All others are String? — null if not returned by the terminal or transport.

Field Type When populated Notes
eventId String Always The same UUIDv4 delivered via onStarted. For a Cloud Pending callback this is the SDK event id; the cloud reference is in transactionId.
status TransactionStatus Always Success, Failed, or Pending (Cloud doTransaction only — delta D23(g)).
billingRefNo String? Request echo Echo of TransactionRequest.billingRefNo. Use this to pair results to requests without a side map.
referenceId String? Request echo Echo of TransactionRequest.referenceId when supplied.
transactionId String? Transport App-to-App: terminal PlutusTransactionLogID. Cloud: PlutusTransactionReferenceID. PADController: terminal-assigned id. Use as the immutable terminal-side identifier in reports.
rrn String? Acquirer Retrieval Reference Number — the acquirer's id for the card transaction.
authCode String? Acquirer The acquirer's authorisation code. Present on approved card transactions.
amount String? Terminal echo String, not numeric — formatting varies by terminal. Compare to your request amount only after parsing.
maskedPan String? Card txn E.g. 411111XXXXXX1111. Never the full PAN.
cardHolderName String? Card data May be padded with whitespace by older terminals; the SDK trims.
cardType String? Terminal E.g. VISA, MASTERCARD, RUPAY. Free-form — do not parse for business logic.
acquirer String? Terminal Bank or scheme acquirer code.
merchantId String? Transport App-to-App: terminal-derived. Cloud / PADController: echoed from request when set. Use to cross-reference back-office records.
terminalId String? Transport App-to-App: terminal-derived. Cloud / PADController: echoed from request when set.
invoiceNo String? Terminal echo App-to-App returns it as a numeric string (e.g. "42").
batchNo String? Terminal The acquirer's batch number for end-of-day settlement.
date String? Terminal Format varies by terminal — commonly YYYYMMDD.
time String? Terminal Format varies — commonly HHmmss, 24-hour.
chargeSlipPrintData String? Transport Content differs per transport — see §02. Never parse generically.
responseCode String? Success path Raw acquirer / terminal response code. App-to-App: "0" on success.
responseMessage String? Success path Raw acquirer / terminal message, e.g. "APPROVED".
metadata Map<String,String>? Transport extras Transport-specific fields the SDK could not promote to typed fields. Keys follow <scope>.<name> convention. See §03.
02

chargeSlipPrintData — what it actually is

The name is historical. The field carries different content on different transports. Treat it as a transport-specific opaque string — never parse it generically.

📱
App-to-App
Contains the Pinelabs PlutusVersion returned by MasterApp (e.g. "1.2.3").

The terminal itself prints the slip — the SDK does not return slip text.
🔌
PADController
A vendor slip body that you can hand to a printer driver if your integration includes one.

Format is terminal-model-dependent.
☁️
Cloud REST
Not populated.

The cloud transport has no slip-printing semantics.
🌐
TCP direct
Defined by the future TCP adapter.

Treat as undefined in v1.
⚠️
The terminal's printer is the canonical slip surface

If you need to print a separate copy of the slip from your own POS, do not assume chargeSlipPrintData is renderable text. On App-to-App it is a version string, not receipt copy.

03

Metadata keys

TransactionResult.metadata collects extras the SDK could not promote to a typed field. Keys follow a <scope>.<name> namespacing convention so future additions don't collide.

🚇
App-to-App — NCMC fields
prefix: ncmc.* · National Common Mobility Card transit transactions
Key Source field Notes
ncmc.serviceID Detail.serviceID Issuer-assigned service id.
ncmc.serviceMI Detail.serviceMI NCMC merchant identifier.
ncmc.isTransitMode Detail.isTransitMode "true" / "false" — boolean rendered as a string for stable cross-language semantics.
ncmc.paymentMode Detail.paymentMode NCMC payment-mode code (free-form pass-through).
ncmc.serviceData Detail.serviceData Free-form NCMC service payload.
ncmc.AppId Detail.AppId NCMC application id.
☁️
Cloud — raw TransactionData
prefix: cloud.transactionData.*
When the cloud server returns its TransactionData object on a status response, every leaf field is flattened under cloud.transactionData.* (delta D23(j)). Null on status = Pending. Keys are stable across releases.
🔌
PADController — stabilising
prefix: pad.<columnName>
PADController returns CSV columns, most of which are promoted to typed fields. Until the column set is locked, any unrecognised column is preserved verbatim under pad.<columnName> to avoid data loss for early adopters.
04

Reading the result

Canonical pattern inside onSuccess:

Kotlin 
override fun onSuccess(result: TransactionResult) {
    val printable = buildString {
        appendLine("Status  : ${result.status}")
        appendLine("Order   : ${result.billingRefNo}")
        result.transactionId?.let  { appendLine("Txn id  : $it") }
        result.rrn?.let            { appendLine("RRN     : $it") }
        result.authCode?.let       { appendLine("Auth    : $it") }
        result.maskedPan?.let      { appendLine("Card    : $it (${result.cardType ?: "—"})") }
        appendLine("Amount  : ${result.amount ?: "?"}")
        appendLine("MID/TID : ${result.merchantId} / ${result.terminalId}")
    }

    // ✅ Persist terminal state before touching UI
    db.markOpCompleted(
        eventId       = result.eventId,
        transactionId = result.transactionId,
        body          = printable
    )

    // ✅ Marshal to UI thread
    runOnUiThread { renderReceipt(printable) }
}

// NCMC check
val isTransit = result.metadata?.get("ncmc.isTransitMode") == "true"
Java 
@Override
public void onSuccess(TransactionResult result) {
    StringBuilder sb = new StringBuilder();
    sb.append("Status  : ").append(result.getStatus()).append('\n');
    sb.append("Order   : ").append(result.getBillingRefNo()).append('\n');
    if (result.getTransactionId() != null)
        sb.append("Txn id  : ").append(result.getTransactionId()).append('\n');
    if (result.getRrn() != null)
        sb.append("RRN     : ").append(result.getRrn()).append('\n');
    if (result.getAuthCode() != null)
        sb.append("Auth    : ").append(result.getAuthCode()).append('\n');
    if (result.getMaskedPan() != null)
        sb.append("Card    : ").append(result.getMaskedPan())
          .append(" (").append(result.getCardType() != null ? result.getCardType() : "—").append(")\n");
    sb.append("Amount  : ").append(result.getAmount() != null ? result.getAmount() : "?").append('\n');
    sb.append("MID/TID : ").append(result.getMerchantId())
      .append(" / ").append(result.getTerminalId());

    String printable = sb.toString();

    // ✅ Persist terminal state before touching UI
    db.markOpCompleted(result.getEventId(), result.getTransactionId(), printable);

    // ✅ Marshal to UI thread
    runOnUiThread(() -> renderReceipt(printable));
}

// NCMC check
Map<String, String> meta = result.getMetadata();
boolean isTransit = meta != null && "true".equals(meta.get("ncmc.isTransitMode"));
05

Forward compatibility

  • ➕ New typed fields may be added to TransactionResult as Pinelabs ops publishes new terminal capabilities. Existing typed fields will not be removed without a major version bump.
  • 🔑 New metadata keys may be added at any time. Existing keys will not change semantics or be removed without a major version bump. Always use a null / absent check before reading any key.
  • 🚫 status will not gain new variants in the v1 line. If Pinelabs introduces a new lifecycle state, it ships in v2. Exhaustive when / switch on TransactionStatus is safe within v1.
