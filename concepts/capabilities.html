TransportOptions × Transport

TransactionRequest.transportOptions is a discriminated union. You must pass the variant matching the active transport or the SDK raises SdkError.InvalidInput synchronously.

App-to-App TransportOptions::AppToApp

All fields are optional: gstIn?, gstBrkUp?, serviceId?, serviceMi?, isTransitMode?, paymentMode?, serviceData?

Cloud TransportOptions::Cloud

Required: transactionNumber, sequenceNumber

Optional: allowedPaymentMode?, autoCancelDurationInMinutes?, forceCancelOnBack?, merchantStorePosCode?

PADController TransportOptions::PadController

Placeholder: extras: Map<String, String>? — concrete typed fields will be promoted from extras as the CSV column set stabilises.

TCP TransportOptions (TCP)

Not yet specified. Supplying a mismatched variant raises SdkError.InvalidInput with a diagnostic detail.

Reading this matrix

Write your code defensively: catch NotSupported and fall back to a transport that does support the capability. This matrix is also the behaviour that the SDK's CI tests guard — what you see is what you get.
