Overview init doTransaction

init

stable Since 0.1.0 lifecycle

Initialize the SDK singleton for the current process.

Description

init constructs the SDK, creates the MasterApp IPC transport, and wires a single-thread worker executor for transaction dispatch. It must be called before any other SDK API.

init takes identity and infrastructure configuration only — merchant appId, operator userId, host app version, and the IPC timeoutSecs. Per-transaction fields belong on each TransactionRequest.

The method is idempotent: a second call with any arguments returns the first-created instance.

📝
init itself does not bind to MasterApp or perform I/O. Binding is lazy per transaction. Transport-related failures surface on doTransaction as TransportUnavailable.

Example

Kotlin 
import com.pinelabs.sdk.PinelabsSdk

val sdk = PinelabsSdk.init(
    context     = applicationContext,
    appId       = "1001",
    userId      = "operator-01",
    version     = "1.0",
    timeoutSecs = 60,
)
